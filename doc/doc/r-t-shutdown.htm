Reference: Tools

sector_shutdown

The sector_shutdown tool allows to gracefully shutdown part or all of the Sector system.

sector_shutdown -a | -i id | -d address(ip:port) | -r rack_topo_path
Parameters
-a
shutdown the complete system.
-i id
shutdown the slave node with this id.
-d address
shutdown the slave node with this ip:port address .
-r rack_topo_path
shutdown a rack. This parameter should be set according to the conf/topology.conf, e.g., /1/1/2.
Description

This command can be used to gracefully shutdown the Sector system or part of it (e.g., to decompose some computers or maintanence). This command requires Sector "root" previlige.
